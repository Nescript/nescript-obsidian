左值(lvalue)是**在内存中有地址的对象**
不是左值的值就是右值(rvalue)，所以我们说**在内存中没有地址的对象是右值**

```cpp
int var; // 声明一个左值
var = 3; // 给左值赋值

(var + 1) = 2; // ERROR! 试图给右值赋值
2 = var; // ERROR! 试图给右值赋值
```

写程序时，我们声明的变量就是左值。声明的过程就是在内存中申请一个位置来存放数据，这就符合了左值的定义。

右值没有地址，只是表达式的临时结果，可能临时存在某个寄存器里面。所以给右值赋值没有意义：根本没有地方装。

# 换个角度看
左值的名字隐含着两个意思，一个是左值放在赋值表达式的左边(left)，另一个是 lvalue 的 l 亦指代 Locator，**有地址的值就是左值。**

右值也是同理，其名字隐含“在赋值表达式右边的东西，语义上最终一定会被转换为右值”。

从这个视角回顾赋值表达式和左右值，可以想象到这样的图景：
- 左值就是容器：箱子、瓶子、罐子等等
- 右值就是要放进容器的物品：杂志、果酱等等
- 赋值表达式就是把物品放进容器的过程。

# 左值和右值的转换
有时候赋值表达式看起来左右都是左值，如下例：
```cpp
a = b;
c = d + e;
```

实际上这里隐含了左值到右值的转换。编译器会先把 b 转换为右值——b 中的物品，再完成赋值：往 a 容器中放入 b 容器内同款的物品。
在逻辑上，**左值到右值转换**的本质是：**“从一个内存地址中读取它存储的具体数值”**。

这样的转换也发生在运算符中：考虑第二行中右侧的 `d + e`，`+` 运算符接收两个右值并返回一个右值，这里就存在左值到右值的隐式转换：d 和 e 这俩左值存储的具体数值被读取并作为右值使用。

所有非数组、非函数或非不完整类型的左值都可以这样转换为右值。
- 数组不行是因为数组名虽是一个左值，存储的却不是一个单一的值，导致逻辑上行不通——要进行右值转换，应该读取数组里的哪个具体数值？
- 函数在代码中也是左值：函数的入口地址是固定的，满足了左值的定义（要区分函数左值和函数返回值，后者往往是右值）。但我们无法从一个函数地址中“读取出一个数值”来作为右值
- 不完整类型是指编译器知道这个类型存在，但不知道它具体占多大内存、包含哪些成员。常见的例子是前置声明：只声明而未定义的类，结构体等等。由于编译器不知道这个类型的内部布局和大小，读取内容自然也是无法做到的。

反过来如何把右值转换成左值呢？答案是不行，我们只可以利用右值“生成”左值。譬如[[一元运算符 `*`]] (解引用)会接受一个右值参数，并返回一个左值(依地址找到容器本身)

与之相反的取地址运算符 `&` 则接受一个左值（容器），返回一个右值（容器所在的位置）。
